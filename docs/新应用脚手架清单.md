# 新应用脚手架清单（可复用模板）

本文是通用模板。每次上新应用时，按清单替换变量并逐项勾选。

## 1. 目录脚手架

目标目录（将 `<appname>` 替换为应用名）：

```text
apphub/
  <appname>/
    app.json
    icon.png
    latest/
      docker-compose.yml
      .env
```

命令：

```powershell
New-Item -ItemType Directory -Force apphub/<appname>/latest
```

## 2. app.json 脚手架（字段清单）

必填最小字段：
1. `appid: -1`
2. `appname: <appname>`
3. `apptitle: <展示名>`
4. `apptype` / `appTypeCN`
5. `appversion`（至少 latest）
6. `appdesc`
7. `appstatus: 1`
8. `home` / `help`
9. `updateat: <unix_timestamp>`
10. `field`（至少包含 domain、allow_access、cpus、memory_limit + 业务字段）
11. `env`（必须包含 app_path、host_ip、cpus、memory_limit + 业务字段）
12. `volumes`

建议 `apptype` 对照：
1. BuildWebsite / 建站
2. Database / 数据库
3. Storage / 存储/网盘
4. Tools / 实用工具
5. Middleware / 中间件
6. AI / AI/大模型
7. Media / 多媒体
8. Email / 邮件/邮局
9. DevOps / DevOps
10. System / 系统

校验命令：

```powershell
Get-Content "apphub/<appname>/app.json" | ConvertFrom-Json | Out-Null
Write-Output "app.json OK"
```

## 3. docker-compose.yml 脚手架

最小模板（按应用改 image/ports/volumes/env）：

```yaml
services:
  <appname>:
    image: <docker_image>
    restart: always
    deploy:
      resources:
        limits:
          cpus: ${CPUS}
          memory: ${MEMORY_LIMIT}
    environment:
      - <APP_ENV_1>=${APP_ENV_1}
      - <APP_ENV_2>=${APP_ENV_2}
    ports:
      - ${HOST_IP}:${APP_PORT}:<container_port>
    volumes:
      - ${APP_PATH}/data:<container_data_path>
    labels:
      createdBy: "bt_apps"
    networks:
      - baota_net

networks:
  baota_net:
    external: true
```

必须保留：
1. `labels.createdBy: "bt_apps"`
2. `cpus/memory` 限制字段
3. `${HOST_IP}` 端口前缀
4. `${APP_PATH}` 数据目录前缀

## 4. .env 脚手架

规则：
1. 变量名全部大写。
2. 值全部留空。
3. 与 `app.json` 的 `env.key`（转大写）严格一致。
4. 必含：`HOST_IP`、`CPUS`、`MEMORY_LIMIT`、`APP_PATH`。

示例：

```env
APP_PORT=
APP_ENV_1=
APP_ENV_2=
HOST_IP=
CPUS=
MEMORY_LIMIT=
APP_PATH=
```

## 5. icon.png 脚手架

要求：
1. PNG。
2. 推荐 100x100。
3. 文件名固定 `icon.png`。

## 6. 上线前检查清单（逐项勾选）

1. [ ] 目录结构正确。
2. [ ] app.json 可解析。
3. [ ] latest 下 `docker-compose.yml` 与 `.env` 同时存在。
4. [ ] `.env` 与 `app.json env` 键名一致。
5. [ ] compose 中包含 `createdBy: "bt_apps"`。
6. [ ] compose 端口使用 `${HOST_IP}`。
7. [ ] compose 卷使用 `${APP_PATH}`。
8. [ ] `updateat` 已更新为当前时间戳。
9. [ ] 图标存在且可显示。
10. [ ] 只保留需要导入的应用目录。

## 7. Git 提交流程脚手架

```powershell
git status --short

# 首次可设置仓库级身份
git config user.name "<github_name>"
git config user.email "<github_noreply_email>"

git add apphub/<appname>
git commit -m "feat: add <appname> apphub template"

git remote set-url origin https://github.com/<user>/apphub.git
git push -u origin main
```

## 8. 宝塔导入与更新脚手架

导入：
1. Docker 应用商店 -> 外部仓库 -> 导入你的仓库地址。
2. 同步/刷新。
3. 安装应用并填写参数。

后续更新：
1. 改模板文件。
2. 更新 `updateat`。
3. 提交推送。
4. 面板同步外部仓库。
5. 需要镜像更新时，执行“更新镜像/重建容器”。

## 9. 常见报错速查

1. 报 `docker-compose.yml 不存在`：通常是 `.env` 缺失。
2. 安装后前端可开、后端连不上：检查 API 路径或应用是否需要首次 `?api=` 引导。
3. 面板看不到更新：`updateat` 未递增或仓库未同步。
4. 容器不受面板托管：compose 未加 `createdBy: "bt_apps"`。
